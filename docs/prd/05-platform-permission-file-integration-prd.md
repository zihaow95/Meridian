# 子 PRD 05：权限、配置、文件、钉钉与集成

版本：V0.1  
日期：2026-06-29  
状态：已确认基线

基线确认日期：2026-07-02
上游依据：总体 PRD、产品全生命周期架构设计说明

## 1. 目标与范围

本模块提供全系统共同依赖的平台能力：

- 用户、组织和钉钉身份认证；
- RBAC + ABAC 权限和专项授权；
- 平台管理权与业务数据访问权分离；
- 模板、字典、属性组和业务参数配置；
- 受控文件和版本；
- 我的待办、站内通知和钉钉通知；
- 审计日志；
- 外部数据接入框架；
- 备份、恢复和基础运行保障。

首期不建设完整多租户运营、在线文档协作或复杂流程设计器。

### 1.1 关键用户故事

- 作为员工，我可以通过钉钉进入系统，并只看到本人权限范围内的信息和待办。
- 作为业务负责人，我可以把任务或专业确认分派给具体人员，而不授予额外全局权限。
- 作为系统管理员，我可以维护角色、模板和集成，但默认不能读取敏感业务内容。
- 作为超级管理员，我可以按开关执行单人管理或双人复核。
- 作为审计人员，我可以追溯关键状态、授权、下载、发布和决策操作。

### 1.2 通用前置条件

- 钉钉企业内部应用、认证回调和必要管理员权限已提供；
- 内部组织、用户标识和角色管理可用；
- 数据库、对象存储、后台任务和审计服务可用；
- 默认权限策略为拒绝，关键角色采用人工授权；
- 外部集成未配置时不影响系统核心业务运行。

## 2. 用户和组织

### 2.1 用户身份

系统内部使用不可变用户标识。钉钉用户标识、姓名、部门、岗位和在职状态作为身份绑定信息，不作为历史业务记录的唯一标识。

用户状态：

- 待启用；
- 正常；
- 停用；
- 离职。

停用或离职用户不能登录和接收新任务，但其历史操作、提案、确认和决策记录永久保留。

### 2.2 组织

- 首期按单一企业组织运行；
- 支持部门和上下级关系；
- 用户可以有主部门，必要时可关联兼任部门；
- 核心业务数据保留组织边界；
- 不建设租户计费、开通和跨租户运营页面。

### 2.3 钉钉同步

钉钉可提供：

- 登录身份；
- 用户基础信息；
- 部门结构；
- 在职状态。

钉钉组织变化不得自动授予产品总监、经管会、经营监督人、系统管理员等关键业务角色。

## 3. 身份认证

### 3.1 登录

- 首期使用钉钉企业内部应用认证；
- 登录成功后绑定或创建内部用户；
- 无法匹配有效用户时拒绝进入并提示联系管理员；
- 停用/离职用户即使钉钉认证成功也不能进入；
- 会话失效后重新认证，不在通知链接中携带长期凭证。

### 3.2 深链接

钉钉通知可跳转到系统具体对象。用户进入后再次执行权限校验；无权限时只显示无权访问，不泄露标题、摘要或文件名。

## 4. 权限模型

### 4.1 RBAC

首期角色至少包括：

- 系统管理员；
- 超级管理员；
- 产品总监；
- 产品经理；
- 提案负责人；
- 项目组长；
- 项目副组长；
- 项目成员；
- 部门负责人；
- 专业确认人；
- 经管会成员；
- 老板/最终决策人；
- 经营监督人；
- 普通员工。

角色定义“可以执行哪类动作”，不单独决定具体业务对象范围。

### 4.2 ABAC

访问判断至少结合：

- 组织和部门；
- 项目身份；
- 产品归属；
- 当前阶段和状态；
- 数据敏感等级；
- 是否联合提案成员；
- 是否当前任务 R、确认人或阶段门处理人；
- 是否存在有效专项授权。

默认拒绝；只有满足角色、对象范围和数据等级要求时才允许。

### 4.3 项目身份

项目组长、副组长、成员、任务 R、专业确认人等是项目实例身份，不应全部固化为全局角色。用户在不同项目中可以拥有不同身份。

### 4.4 数据敏感等级

| 等级 | 示例 |
|---|---|
| 公开摘要 | 已立案项目标题和公开摘要 |
| 内部可见 | 已上市产品基本信息 |
| 项目受控 | 项目计划、一般交付物 |
| 敏感受控 | 成本、供应商、工艺、未上市包装 |
| 高敏受控 | 核心配方、重大经营决策材料 |

### 4.5 动作权限

权限必须分别控制：

- 页面和对象查看；
- 字段/属性组查看和编辑；
- 文件预览、下载和上传；
- 导出；
- 分享；
- 通知摘要；
- 任务分派；
- 专业确认；
- 阶段门处理；
- 角色和专项授权。

“能看到页面”不代表能看到所有字段或下载所有文件。

## 5. 关键角色和授权

### 5.1 人工配置

关键角色由系统管理员按已批准名单人工配置，不根据钉钉岗位或职级自动授予。每次变更记录配置人、时间、前后角色和生效范围。

### 5.2 超级管理员双人复核

系统提供开关：

- 关闭：指定超级管理员可以单人执行关键平台管理；
- 开启：关键超级管理员变更需要第二名授权人员复核。

开关本身属于关键配置，变更必须留痕。双人复核只覆盖平台管理动作，不替代业务审批。

### 5.3 专项授权

专项授权至少包含：

- 被授权人；
- 业务对象或范围；
- 允许动作；
- 数据等级；
- 生效和失效时间；
- 授权人；
- 状态。

到期自动失效。专项授权不能授予授权人本人不具备的权限。

### 5.4 排障访问

平台管理员默认不能查看敏感业务内容。排障访问必须：

- 明确对象和时间范围；
- 限时有效；
- 记录申请/开启人；
- 全量记录访问和下载动作；
- 到期自动关闭。

首期可由具备权限的超级管理员开启，不建设独立工单系统。

## 6. 模板与配置中心

### 6.1 配置范围

首期支持维护：

- 生命周期阶段模板；
- 部门责任模板；
- 任务模板；
- 三层交付物模板；
- 专业确认要求；
- 产品档案属性组和字段；
- 指标、阈值和数据映射；
- 数据敏感等级映射；
- 提案额度和联合成员人数；
- 通知规则；
- 字典和渠道。

### 6.2 版本规则

- 配置草稿可以反复编辑；
- 发布后形成不可变版本；
- 新项目使用发布时的模板快照；
- 修改模板只影响新创建项目；
- 历史项目不因模板发布自动变化；
- 项目若主动升级模板，必须展示差异并由产品总监确认；首期不强制提供批量升级。

### 6.3 配置权限

- 平台结构配置由系统管理员维护；
- 业务内容由产品总监或指定业务管理员确认；
- 系统管理员不能借配置权限读取敏感业务值；
- 删除已被引用的配置只允许停用。

## 7. 文件和版本管理

### 7.1 文件对象

文件至少记录：

- 文件名、类型和大小；
- 所属业务对象；
- 文档分类；
- 版本号；
- 状态；
- 敏感等级；
- 上传人和时间；
- 校验值；
- 受控/作废状态；
- 来源：项目上传、产品档案、迁移历史、外部同步。

### 7.2 版本状态

```text
草稿 → 已提交/锁定 → 受控
                    └─ 退回
受控 → 新版本修订
受控 → 作废
```

- 提交后当前版本不可直接覆盖；
- 阶段门和专业确认引用具体版本；
- 作废不物理删除；
- 同一文件的新版本保留版本链；
- 历史决策引用版本永久可追溯。

### 7.3 上传

- 上传前校验大小、类型和空文件；
- 上传失败不创建业务版本；
- 同名文件不自动覆盖；
- 文件存储与业务数据库使用稳定关联标识；
- 首期支持常用办公文档、图片和压缩包，具体类型由管理员配置。

### 7.4 预览、下载和导出

- 预览与下载分别授权；
- 高敏文件可配置禁止下载；
- 下载和批量导出必须审计；
- 通知中不附带受控文件，只提供系统链接；
- 分享不能突破接收人原有权限。

## 8. 待办和通知

### 8.1 待办来源

- 提案补充和评审；
- 立案评估；
- 任务和交付物；
- 专业确认；
- 阶段门；
- 暂缓池季度回看；
- 经营议题；
- 配置或授权复核。

### 8.2 系统待办

系统待办是唯一权威处理入口，至少显示标题、来源、对象、截止时间、状态和深链接。任务完成或业务状态变化后自动关闭或更新待办。

### 8.3 钉钉通知

首期钉钉只做：

- 身份认证；
- 消息通知；
- 深链接。

通知摘要必须按接收人权限生成。钉钉不承载确认、审批或状态变更。

### 8.4 重复和失败

- 相同对象和动作的重复通知应合并或抑制；
- 通知发送失败记录错误并允许后台重试；
- 发送失败不回滚业务操作；
- 用户仍能在系统待办中看到权威状态。

## 9. 审计

### 9.1 审计范围

至少记录：

- 登录和身份绑定；
- 角色、权限和专项授权；
- 查看高敏内容；
- 文件下载和导出；
- 状态流转；
- 任务和负责人变更；
- 专业确认和阶段门；
- 产品档案草稿和发布；
- 人工经营数据修正；
- 配置发布；
- 排障访问；
- 先执行后补确认。

### 9.2 审计字段

- 操作人和代办/授权身份；
- 时间；
- 对象类型和标识；
- 动作；
- 操作前后值或摘要；
- 来源终端/请求标识；
- 成功或失败；
- 关联业务记录。

审计日志只追加，不允许普通管理员编辑或删除。

## 10. 外部集成框架

### 10.1 数据主权

- 本系统负责产品定义、生命周期治理、产品档案和受控决策；
- ERP、MES、WMS、销售/BI 和财务系统继续负责执行事实；
- 外部数据保存来源、外部标识和同步时间；
- 决策时保存不可变快照。

### 10.2 接入方式

首期支持：

- API 适配器；
- 定时任务；
- Excel/CSV；
- 手工维护。

每个接入任务必须记录批次、状态、数量、错误和重试信息。

### 10.3 失败处理

- 外部系统不可用时不删除已有有效数据；
- 同步失败可重试；
- 部分失败单独记录错误行/对象；
- 不可识别映射进入待处理区；
- 接口调用和凭据错误不得暴露给普通业务用户。

## 11. 运行保障

### 11.1 容量

- 约 200 名员工；
- 约 10 个并行项目；
- 受控文件约 100GB/年。

### 11.2 备份与恢复

- 数据库每日备份；
- 文件存储每日备份；
- 备份位于不同于生产的位置；
- 定期执行恢复验证；
- 首期 RPO ≤ 24 小时；
- 首期 RTO ≤ 24 小时。

备份任务失败必须通知系统管理员并在运行看板显示。

### 11.3 可追溯性

历史决策引用的文件版本、产品档案版本、数据快照和审计记录不得因后续修改而变化。

## 12. 管理页面

### 12.1 用户与角色

支持查看用户状态、组织、钉钉绑定、全局角色和有效专项授权。

### 12.2 权限检查

授权管理员可输入用户和业务对象，查看最终允许/拒绝结果及命中的策略摘要，便于排障，但不展示系统机密或其他用户无关数据。

### 12.3 配置中心

按配置类型展示草稿、已发布、停用和版本历史；发布前显示影响范围。

### 12.4 集成与运行

展示同步批次、通知失败、备份状态、存储容量和最近恢复验证结果。

## 13. 异常与边界

- 用户离职时不删除历史记录，未完成任务进入待重新分派列表；
- 部门负责人变更不自动改写历史专业确认；
- 角色被撤销后立即影响新请求，已打开页面再次操作时重新校验；
- 专项授权到期后下载链接立即失效；
- 配置发布失败保持原发布版本有效；
- 文件与业务记录关联失败时删除未完成临时上传，不产生孤立受控版本；
- 审计写入失败时关键业务操作必须失败并明确提示，不能无审计继续；
- 钉钉通知失败不影响系统内业务结果。

## 15. 验收场景

### 15.1 钉钉登录但无角色

用户通过钉钉认证后可以作为普通员工进入系统，但不会因职位是总监而自动获得产品总监或经管会权限。

### 15.2 平台管理员隔离

系统管理员可以配置“配方”属性和文件类型，但不能查看具体产品的高敏配方或下载高敏文件。

### 15.3 专项授权到期

用户在授权期内可查看指定项目文件；到期后页面、下载和通知摘要均重新受限，历史访问记录仍可审计。

### 15.4 模板发布

模板 V2 发布后，新项目使用 V2；已创建项目继续使用 V1 快照，任务和交付物不被自动改写。

### 15.5 文件版本

阶段门引用文件 V1 后上传 V2，历史决策继续指向 V1；V2 未重新专业确认前不能替换当前阶段门引用。

### 15.6 通知失败

钉钉发送失败时，业务操作成功且系统待办存在；后台记录失败并重试，不产生重复业务动作。

### 15.7 审计失败

关键阶段门操作无法写入审计时，系统拒绝提交并明确提示，不能形成无审计决策。

### 15.8 恢复验证

管理员可以查看最近数据库和文件备份、恢复验证时间及结果；备份失败触发站内告警。
